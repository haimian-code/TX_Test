# 设计文档：余烬档案馆流派验证器

## 1. 世界观与玩法循环

玩家扮演“档案清道夫”，进入被失控魔导记录污染的余烬档案馆。每次模拟代表一次清理行动：玩家根据外部策略先分配属性点、选择道具倾向，再按回合制或 tick 制战斗规则依次击败关卡队列中的怪物。模拟器记录胜负、耗时、伤害曲线和技能释放次数，用于验证不同流派在同一关卡下的强弱。

本项目不是完整商业游戏，而是面向策划和 AI 生成内容的轻量验证工具。它的目标是快速回答：“这套配置下，某个 build 是否可行，是否比另一个 build 更稳定？”

## 2. 角色系统

当前示例只有 1 个可玩角色：

| 字段 | 说明 |
| --- | --- |
| 名称 | 档案清道夫 |
| 基础 HP | 120 |
| 攻击 | 12 |
| 防御 | 3 |
| 暴击率 | 8% |
| 暴击伤害 | 180% |
| 速度 | 10 |
| 能量恢复 | 6 / 秒 |

开局有 6 点属性点，由策略 API 分配。当前支持加点字段：

| 字段 | 每点收益 | 用途 |
| --- | --- | --- |
| attack | +2 攻击 | 提升直接伤害和持续伤害基数 |
| defense | +1 防御 | 降低受到的普攻伤害 |
| crit | +4% 暴击率 | 提高爆发技能收益 |
| speed | +1 速度 | 影响回合制先攻和 tick 制攻击节奏 |
| max_hp | +12 最大 HP | 提高容错 |

## 3. 技能系统

角色拥有 3 个技能，技能释放完全由策略 API 决定。

| 技能 | 冷却 | 消耗 | 效果 | 策略意义 |
| --- | --- | --- | --- | --- |
| 迅捷斩 | 1.2s | 8 能量 | 造成 1.0 倍攻击直接伤害 | 低成本填充技能 |
| 裂创标记 | 3.0s | 14 能量 | 造成 1.85 倍攻击直接伤害，额外 +16% 暴击率 | 暴击爆发流核心 |
| 余烬绽放 | 4.0s | 18 能量 | 施加 4 秒持续伤害，每秒 0.45 倍攻击 | 持续输出流核心 |

技能释放时机会影响强弱：爆发技能需要在能量足够时尽早打出以缩短战斗；持续伤害技能需要保持覆盖率，过晚释放会损失 tick 数。

## 4. 道具系统

| 道具 | 效果 | 组合潜力 |
| --- | --- | --- |
| 剃刀透镜 | 暴击率 +12%，暴击伤害 +25% | 与战斗手册组合后，高频技能放大暴击收益 |
| 腐蚀瓶 | 持续伤害 +35% | 与战斗手册组合后，余烬绽放覆盖率提高 |
| 战斗手册 | 冷却 -12%，能量恢复 +2 | 是两个流派的通用加速件 |

道具由配置定义，运行时由策略选择启用顺序。当前示例简化为玩家拥有 3 件道具，策略根据流派偏好启用其中的核心道具。

当前模拟器已经实现轻量随机词缀：每局会根据随机种子给策略选中的每件道具 roll 一个词缀，词缀会修改角色属性，并记录到单局结果和批量统计中。词缀池包括锋锐、凝神、过载、蚀刻、稳固，用于体现肉鸽 build 的随机扰动。

## 5. 怪物系统

| 怪物 | 定位 | 特征 |
| --- | --- | --- |
| 纸页仆从 | 普通怪 | HP 低、攻击频率高，用于测试清杂效率 |
| 墨痕守卫 | 精英怪 | HP 高、防御高、攻击慢但重，用于测试长战稳定性 |

怪物 AI 当前为固定普攻。题目不要求复杂 AI，本项目把复杂度集中在策略 API 和结果统计。

## 6. 关卡结构

示例关卡“档案馆验证场”按队列刷怪：

1. 纸页仆从 x3
2. 墨痕守卫 x1
3. 纸页仆从 x2

同一时间只与队列头部怪物战斗。当前怪死亡后立即切换下一只怪物，直到玩家死亡或怪物队列清空。

## 7. 流派假设

### 7.1 暴击爆发流

策略倾向：

- 属性点优先 attack / crit / speed
- 道具优先剃刀透镜 + 战斗手册
- 技能优先裂创标记，其次迅捷斩

预期：对单体精英怪更强，平均耗时短，但胜率会受到暴击随机性的影响。

### 7.2 腐蚀持续流

策略倾向：

- 属性点优先 attack / defense / max_hp
- 道具优先腐蚀瓶 + 战斗手册
- 技能优先余烬绽放，保持持续伤害覆盖，其次迅捷斩

预期：胜率更稳定，尤其在长战斗中更安全；但击杀普通怪可能慢于暴击流。

## 8. 验证命题

本模拟器要验证的问题是：

- 在“档案馆验证场”中，暴击爆发流是否拥有更短平均耗时？
- 腐蚀持续流是否拥有更高或更稳定的胜率？
- 如果实际结果与预期不符，能否通过日志、伤害曲线和技能释放次数解释原因？

README 的设计自评会基于批量模拟结果回答这些问题。
